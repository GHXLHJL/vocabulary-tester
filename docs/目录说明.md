# 项目目录说明

## 根目录
- `index.html`、`app.js`、`style.css`
  - 前端网页主文件
  - 为了兼容静态站点部署，继续保留在根目录
- `一键更新部署.bat`
  - 用户主入口
  - 负责从词库 txt 更新网页内容并推送 GitHub
- `统计单词数量.bat`
  - 用户检测入口
  - 统计词库中的英文单词数量
- `检测单词集问题.bat`
  - 用户检测入口
  - 检测重复单词和疑似拼写问题

## scripts
- `scripts/node`
  - 存放 Node.js 内部辅助脚本
  - `replace_words_helper.js`: 将 txt 词库写入前端并自动更新版本号
  - `add_word_helper.js`: 保留的辅助更新脚本
- `scripts/python`
  - 存放 Python 检测脚本
  - `count_words_in_txt.py`: 统计单词数量
  - `check_vocabulary_txt.py`: 检查重复词和疑似拼写问题

## cache
- 存放脚本运行中产生的缓存文件
- `.english_words_cache.txt`: 通用英文公共词库缓存
- `.kaoyan_words_cache.txt`: 考研公共词库缓存
- `.spellcheck_cache.json`: 旧版拼写检测遗留缓存

## .trae
- IDE / 系统相关目录
- `.trae/specs/vocabulary-tester/相似单词集.txt` 是当前词库源文件
- `checklist.md`、`spec.md`、`tasks.md` 属于规格与任务文件，建议不要随意移动

## docs
- 存放说明文档与整理记录
- `目录说明.md`
  - 当前项目目录分类说明
- `工作记录/`
  - 存放本次整理过程中生成的计划、发现、进度记录

## 维护建议
- 日常使用时：
  - 双击根目录下的 `.bat` 文件即可
  - 不需要直接进入 `scripts/`
- 如果以后继续扩展：
  - 新的内部脚本优先放进 `scripts/`
  - 新的缓存文件优先放进 `cache/`
  - 新的说明材料优先放进 `docs/`
