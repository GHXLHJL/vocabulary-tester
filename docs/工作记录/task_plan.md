# 任务计划

## 目标
- 为项目建立清晰的目录分类。
- 保持网页前端和双击脚本可继续正常使用。
- 为后续维护增加目录说明文档。

## 分类原则
- 根目录保留网页发布必需文件和用户最常双击的入口脚本。
- 工具脚本集中到 `scripts/`。
- 缓存文件集中到 `cache/`。
- 说明与结构文档集中到 `docs/`。
- `.trae/` 视为 IDE/系统相关目录，不移动。

## 阶段
- [complete] 盘点现有文件与路径依赖
- [complete] 重组脚本与缓存目录
- [complete] 新增说明文档
- [complete] 运行验证并收尾

## 风险
- `index.html`、`app.js`、`style.css` 需要保留在根目录，避免影响静态站点部署。
- 根目录 `.bat` 入口若移动，双击体验会变差，因此优先保留入口、改内部调用路径。

## 错误记录
- `session-catchup.py` 路径不存在，未能使用技能自带恢复脚本；改为手动盘点当前项目结构。
