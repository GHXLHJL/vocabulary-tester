## 🧰 基础安全工具使用指南

熟练掌握Nmap和Sqlmap的常用命令是实习生的基本功。

### 1. Nmap (端口扫描神器)

**场景**：拿到一个IP，想知道它开了哪些端口、运行什么服务。

*   **常用命令 (背诵)**:
    *   `nmap -sS <ip>`: 半连接扫描 (TCP SYN)，速度快且隐蔽（不建立完整TCP连接）。
    *   `nmap -sV <ip>`: 探测端口服务的具体版本 (Service Version)。
    *   `nmap -p- <ip>`: 扫描全端口 (1-65535)，默认只扫Top 1000。
    *   `nmap -O <ip>`: 探测操作系统类型 (OS Detection)。
    *   `nmap -A <ip>`: 综合扫描（包含版本、OS、脚本等）。
*   **高级用法**:
    *   `nmap -sC <ip>`: 使用默认脚本扫描。
    *   `nmap --script vuln <ip>`: 扫描已知漏洞。
    *   `nmap -T4 <ip>`: 加快扫描速度（T0-T5，T5最快）。
    *   `nmap -oN output.txt <ip>`: 将结果保存到文件。

### 2. Sqlmap (自动化注入神器)

**场景**：手工确认有注入后，用它跑数据库数据。

*   **常用命令 (背诵)**:
    *   **基础跑库**:
        `python sqlmap.py -u "http://xxx.com?id=1" --dbs` (列出所有数据库名)
    *   **跑表名**:
        `... -D test_db --tables` (列出test_db库里的表)
    *   **跑列名**:
        `... -D test_db -T users --columns` (列出users表里的列)
    *   **跑数据**:
        `... -D test_db -T users -C "user,pass" --dump` (导出数据)
*   **高阶用法**:
    *   `--batch`: 自动回答Yes，不需人工交互。
    *   `--cookie "PHPSESSID=xxx"`: 如果需要登录，带上Cookie。
    *   `--os-shell`: 尝试获取系统交互式Shell (最高权限)。
    *   `--tamper=space2comment`: 使用tamper脚本绕过WAF。
    *   `--level=3 --risk=1`: 提高测试等级和风险等级。
    *   `--proxy=http://127.0.0.1:8080`: 使用代理。

### 3. Burp Suite (Web 代理/抓包)

*   **核心模块**:
    *   **Proxy (代理模块)**: 拦截HTTP/HTTPS请求和响应。
    *   **Repeater (重放模块)**: 修改数据包并重新发送，测试逻辑漏洞、注入等。
    *   **Intruder (爆破模块)**: 对特定参数（如密码、验证码）进行字典攻击。
    *   **Decoder (编码/解码模块)**: 对数据进行各种编码解码。
    *   **Comparer (比较模块)**: 比较两个数据包的差异。
*   **使用流程**:
    1.  配置浏览器代理为127.0.0.1:8080。
    2.  访问目标网站，Burp会拦截请求。
    3.  将请求发送到Repeater进行测试。
    4.  使用Intruder进行暴力破解。

### 5. 防火墙与WAF配置

#### Windows防火墙配置
```powershell
# 查看所有防火墙规则
# Get-NetFirewallRule: 获取防火墙规则
# Select-Object: 选择特定属性显示
# DisplayName: 规则显示名称
# Direction: 规则方向（Inbound入站/Outbound出站）
# Action: 规则动作（Allow允许/Block阻止）
Get-NetFirewallRule | Select-Object DisplayName, Direction, Action

# 添加入站规则（允许特定端口）
# New-NetFirewallRule: 创建新的防火墙规则
# -DisplayName: 规则显示名称
# -Direction Inbound: 入站规则（从外网进入内网）
# -Protocol TCP: 协议类型为TCP
# -LocalPort 80: 本地端口为80（HTTP端口）
# -Action Allow: 动作为允许
New-NetFirewallRule -DisplayName "Allow HTTP" -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow

# 添加出站规则（阻止特定IP）
# -Direction Outbound: 出站规则（从内网访问外网）
# -RemoteAddress: 远程地址（目标IP）
# -Action Block: 动作为阻止
New-NetFirewallRule -DisplayName "Block Malicious IP" -Direction Outbound -RemoteAddress "1.2.3.4" -Action Block

# 删除规则
# Remove-NetFirewallRule: 删除防火墙规则
# -DisplayName: 指定要删除的规则名称
Remove-NetFirewallRule -DisplayName "Old Rule"
```

#### Linux防火墙配置 (iptables)
```bash
# 查看当前规则
# iptables: Linux防火墙工具
# -L: 列出规则
# -n: 数字格式显示（不解析IP地址）
# -v: 详细模式（显示更多信息）
iptables -L -n -v

# 添加规则（允许SSH）
# -A INPUT: 在INPUT链末尾添加规则
# -p tcp: 协议为TCP
# --dport 22: 目标端口为22（SSH端口）
# -j ACCEPT: 动作为接受（允许）
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# 添加规则（阻止特定IP）
# -s 1.2.3.4: 源地址为1.2.3.4
# -j DROP: 动作为丢弃（拒绝）
iptables -A INPUT -s 1.2.3.4 -j DROP

# 保存规则
# iptables-save: 将当前规则保存到标准输出
# > /etc/iptables/rules.v4: 重定向到文件（IPv4规则文件）
iptables-save > /etc/iptables/rules.v4
```

#### WAF规则配置
```yaml
# ModSecurity规则示例
# SecRule: ModSecurity规则指令
# REQUEST_URI: 检查请求URI
# @rx: 正则表达式匹配
# \.php$: 匹配以.php结尾的URI
# id:1001: 规则ID
# phase:1: 在第1阶段检查（请求头阶段）
# deny: 拒绝请求
# status:403: 返回403状态码
# msg: 拒绝消息
SecRule REQUEST_URI "@rx \.php$" \
    "id:1001,phase:1,deny,status:403,msg:'PHP files not allowed'"

# ARGS: 检查所有请求参数
# union.*select: 匹配包含union和select的SQL注入特征
# phase:2: 在第2阶段检查（请求体阶段）
SecRule ARGS "@rx union.*select" \
    "id:1002,phase:2,deny,status:403,msg:'SQL Injection detected'"

# <script>: 匹配包含script标签的XSS特征
SecRule ARGS "@rx <script" \
    "id:1003,phase:2,deny,status:403,msg:'XSS detected'"
```

#### WAF绕过思路
*   **大小写混合**: `UnIoN SeLeCt` (针对老旧WAF)。
    *   *💡 解释：通过大小写混合绕过基于字符串匹配的WAF规则。*
*   **编码绕过**: URL双重编码 (`%2527` -> `%27` -> `'`)。
    *   *💡 解释：通过URL编码绕过WAF的字符过滤，双重编码可以绕过单层解码的WAF。*
*   **HTTP参数污染 (HPP)**: `?id=1&id=2` (WAF检第一个，后端用第二个)。
    *   *💡 解释：通过重复参数名，让WAF只检查第一个参数，而后端使用第二个参数。*
*   **注释符绕过**: 使用 `--`, `#`, `/**/` 等注释符。
    *   *💡 解释：使用SQL注释符绕过WAF的关键词检测，将恶意SQL语句拆分或注释。*

```cmd
reg import backup.reg
```

#### PowerShell高级操作
```powershell
# 获取进程详细信息
# Get-Process: 获取所有进程
# Where-Object: 筛选条件
# ProcessName -like "*chrome*": 进程名包含chrome
# Select-Object: 选择特定属性
# Id: 进程ID
# ProcessName: 进程名
# CPU: CPU使用时间
# WorkingSet: 工作集（内存使用）
Get-Process | Where-Object {$_.ProcessName -like "*chrome*"} | Select-Object Id, ProcessName, CPU, WorkingSet

# 获取服务详细信息
# Get-Service: 获取所有服务
# Status -eq "Running": 状态为运行中
# Name: 服务名称
# DisplayName: 服务显示名称
# StartType: 启动类型（自动、手动等）
Get-Service | Where-Object {$_.Status -eq "Running"} | Select-Object Name, DisplayName, StartType

# 监控文件系统变化
# New-Object: 创建新对象
# System.IO.FileSystemWatcher: 文件系统监视器
# $watcher.Path = "C:\temp": 监控路径
# EnableRaisingEvents = $true: 启用事件触发
# Register-ObjectEvent: 注册对象事件
# EventName Created: 文件创建事件
# SourceIdentifier: 事件源标识符
# -Action: 事件触发时执行的动作
$watcher = New-Object System.IO.FileSystemWatcher
$watcher.Path = "C:\temp"
$watcher.EnableRaisingEvents = $true
Register-ObjectEvent -InputObject $watcher -EventName Created -SourceIdentifier FileSystem -Action {
    Write-Host "File created: $($event.SourceArgs.FullPath)"
}
```

#### Git基本概念
*   **仓库 (Repository)**: 存储代码的地方，包含所有版本历史。

*   **提交 (Commit)**: 保存代码更改的操作，每个提交都有唯一的哈希值。
    *   *💡 解释：提交是Git中的基本操作单元，记录了代码在某个时间点的状态，可以通过哈希值唯一标识。*
*   **分支 (Branch)**: 代码的并行版本，用于开发新功能或修复bug。
    *   *💡 解释：分支允许在不影响主代码的情况下进行开发，每个分支都是独立的代码版本。*
*   **远程仓库 (Remote)**: 位于网络或其他位置的仓库副本（如GitHub、GitLab）。
    *   *💡 解释：远程仓库是存储在服务器上的Git仓库，用于团队协作和代码备份。*

#### Git常用命令
*   **初始化与克隆**:
    *   `git init`: 在当前目录初始化一个新的Git仓库。

    *   `git clone <url>`: 克隆远程仓库到本地。
        *   *💡 解释：从远程仓库下载完整的项目代码和版本历史到本地。*
*   **基本操作**:
    *   `git add <file>`: 将文件添加到暂存区。
        *   *💡 解释：将文件的修改添加到暂存区，准备提交。*
    *   `git commit -m "commit message"`: 提交暂存区的更改。
        *   *💡 解释：将暂存区的更改保存到仓库历史，需要提供提交信息说明更改内容。*
    *   `git status`: 查看仓库当前状态。
        *   *💡 解释：查看工作区、暂存区的文件状态，了解哪些文件被修改、添加或删除。*
    *   `git log`: 查看提交历史。
        *   *💡 解释：查看仓库的提交历史，包括提交哈希、作者、日期、提交信息等。*
*   **分支管理**:
    *   `git branch`: 查看所有分支。
        *   *💡 解释：列出所有本地分支，当前分支前会有*标记。*
    *   `git checkout -b <branch>`: 创建并切换到新分支。
        *   *💡 解释：创建一个新分支并切换到该分支，相当于git branch和git checkout的组合。*
    *   `git merge <branch>`: 合并指定分支到当前分支。
        *   *💡 解释：将指定分支的更改合并到当前分支，可能需要解决冲突。*
*   **远程操作**:
    *   `git remote -v`: 查看远程仓库地址。
        *   *💡 解释：查看配置的远程仓库URL，包括fetch和push地址。*
    *   `git push origin <branch>`: 推送本地分支到远程仓库。
        *   *💡 解释：将本地分支的提交推送到远程仓库，更新远程仓库的代码。*
    *   `git pull origin <branch>`: 从远程仓库拉取更新。
        *   *💡 解释：从远程仓库获取最新代码并合并到本地分支，相当于git fetch和git merge的组合。*
*   **撤销操作**:
    *   `git checkout -- <file>`: 撤销工作区的修改。
        *   *💡 解释：丢弃工作区中文件的修改，恢复到上次提交的状态（危险操作）。*
    *   `git reset HEAD <file>`: 撤销暂存区的修改。
        *   *💡 解释：将文件从暂存区移除，但保留工作区的修改。*
    *   `git reset --hard <commit>`: 回退到指定提交（危险操作）。
        *   *💡 解释：将仓库、暂存区、工作区都回退到指定提交，丢弃之后的所有更改（非常危险）。*

#### Git安全最佳实践
*   **不要提交敏感信息**: 如密码、API密钥等。

    *   *💡 解释：定期从远程仓库拉取最新代码，可以减少合并冲突的发生。*

```bash
#!/bin/bash
# 监控系统资源使用情况

while true; do
    echo "=== $(date) ==="
    
    # CPU使用率
    # top: 显示系统进程和资源使用
    # -bn1: 批处理模式，运行1次，不显示头信息
    # grep "Cpu(s)": 过滤CPU行
    # awk '{print $2}': 提取第2列（用户CPU使用率）
    # cut -d'%' -f1: 以%为分隔符，取第1列
    cpu_usage=$(top -bn1 | grep "Cpu(s)" | awk '{print $2}' | cut -d'%' -f1)
    echo "CPU Usage: ${cpu_usage}%"
    
    # 内存使用率
    # free: 显示内存使用情况
    # grep Mem: 过滤内存行
    # awk: 计算内存使用率
    mem_usage=$(free | grep Mem | awk '{printf("%.2f", $3/$2 * 100.0)}')
    echo "Memory Usage: ${mem_usage}%"
    
    # 磁盘使用率
    # df: 显示磁盘使用情况
    # -h: 人类可读格式
    # awk 'NR==2{print $5}': 第2行，第5列（使用率）
    disk_usage=$(df -h / | awk 'NR==2{print $5}')
    echo "Disk Usage: ${disk_usage}"
    
    sleep 60
done
```
