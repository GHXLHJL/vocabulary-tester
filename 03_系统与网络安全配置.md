# 系统与网络安全配置

这份文档涵盖了Windows/Linux系统、数据库、中间件的安全加固（基线检查）、防火墙与WAF配置、Linux常用命令、数据安全技术以及常见安全设备认知，是安服工程师必备的系统安全知识。

## 🔐 安全配置与基线加固 (面试核心)

JD中明确要求"熟悉安全配置"，这通常指**基线检查 (Baseline Check)**。面试时挑几个重点说。

### 1. 操作系统加固 (Windows/Linux)

#### Windows系统加固
*   **账号口令**:
    *   强制密码复杂度（大小写+数字+特殊字符）。
    *   设置定期修改周期（90天）。
    *   启用密码历史记录，防止重复使用旧密码。
*   **账户锁定**:
    *   连续输错5次密码锁定账户30分钟（防爆破）。
    *   设置锁定时间阈值和重置时间。
*   **最小权限**:
    *   严禁业务程序（如Tomcat）直接用system权限运行。
    *   普通用户不要加入管理员组。
*   **补丁管理**:
    *   及时更新系统安全补丁。
    *   定期检查未打补丁的编号（如KBxxxx）。
*   **审计策略**:
    *   启用登录审计（成功/失败）。
    *   启用对象访问审计。
    *   启用策略更改审计。

#### Linux系统加固
*   **账号安全**:
    *   禁用不必要的系统账号。
    *   设置root密码复杂度。
    *   限制su命令的使用（只允许特定用户）。
*   **密码策略**:
    *   设置密码过期时间（`chage -M 90 username`）。
    *   设置密码最小长度和复杂度（`/etc/login.defs`）。
*   **SSH加固**:
    *   禁止root直接登录（`PermitRootLogin no`）。
    *   修改默认端口（`Port 2222`）。
    *   只允许密钥登录（`PasswordAuthentication no`）。
    *   限制登录IP（`AllowUsers user@192.168.1.100`）。
*   **文件权限**:
    *   关键文件权限：`/etc/passwd` 644, `/etc/shadow` 600。
    *   用户主目录权限：700或755。
*   **日志审计**:
    *   启用系统日志（`/var/log/secure`, `/var/log/messages`）。
    *   配置日志轮转（`logrotate`）。
    *   定期检查异常登录（`last`, `lastb`）。

### 2. 数据库加固

#### MySQL加固
*   **禁止外网连接**:
    *   `bind-address = 127.0.0.1`。
    *   如果需要外网访问，使用防火墙限制IP。
*   **删除默认库**:
    *   删除 `test` 库。
    *   删除匿名用户。
*   **文件权限**:
    *   限制 `secure_file_priv` 防止任意文件读写。
    *   设置 `secure_file_priv = /var/lib/mysql-files`。
*   **用户权限**:
    *   删除默认的root空密码。
    *   为每个应用创建独立的数据库用户。
    *   遵循最小权限原则。
*   **配置安全**:
    *   禁用 `LOAD DATA INFILE`。
    *   禁用 `LOCAL INFILE`。
    *   启用查询日志审计。

#### Redis加固 (重灾区)
*   **设置密码**:
    *   `requirepass <复杂密码>`（默认是空密码）。
    *   定期更换密码。
*   **重命名高危命令**:
    *   `rename-command FLUSHALL ""`。
    *   `rename-command CONFIG ""`。
    *   `rename-command EVAL ""`。
*   **绑定内网**:
    *   `bind 127.0.0.1`。
    *   如果需要外网访问，使用防火墙限制IP。
*   **禁用危险功能**:
    *   禁用 `CONFIG` 命令。
    *   禁用 `FLUSHDB` 和 `FLUSHALL` 命令。

### 3. 中间件加固

#### Nginx/Apache加固
*   **隐藏版本号**:
    *   防止攻击者根据版本找漏洞 (`server_tokens off`).
    *   修改HTTP响应头中的Server字段。
*   **禁止目录浏览**:
    *   防止源码泄露 (`autoindex off`).
    *   配置默认首页（index.html, index.php）。
*   **限制HTTP方法**:
    *   只允许GET、POST、HEAD方法。
    *   禁用PUT、DELETE等危险方法。
*   **限制请求大小**:
    *   防止DoS攻击。
    *   设置 `client_max_body_size` 限制上传文件大小。
*   **配置安全头**:
    *   `X-Frame-Options: DENY`（防点击劫持）。
    *   `X-XSS-Protection: 1; mode=block`（防XSS）。
    *   `Content-Security-Policy`（内容安全策略）。

#### Tomcat加固
*   **删除默认后台**:
    *   删除 `webapps/ROOT` 下的默认管理页面（manager app）。
    *   防止弱口令上传War包。
*   **修改默认端口**:
    *   8080改为其他端口。
    *   8005（AJP端口）改为其他或禁用。
*   **用户管理**:
    *   删除 `tomcat-users.xml` 中的默认用户。
    *   为管理员设置强密码。
*   **禁用AJP连接器**:
    *   如果不需要，禁用AJP协议（Ghostcat漏洞）。
*   **错误页面**:
    *   配置自定义错误页面。
    *   防止信息泄露。

## 🐧 Linux 常用命令 (安全应急必会)

面试中如果说"熟悉操作系统"，默认包含Linux。

### 1. 进程与网络
*   `ps -ef`: 查看所有进程 (类似Windows的tasklist)。
    *   `[终端: Linux Bash]` *💡 提示：常配合grep使用，如 `ps -ef | grep java` 查看Java进程。*
*   `netstat -antlp`: 查看TCP端口及对应进程PID (最常用)。
    *   `[终端: Linux Bash]` *💡 解释：-a(all) -n(数字形式) -t(TCP) -l(listen状态) -p(显示进程名/PID)。*
*   `top`: 实时查看资源占用（查挖矿病毒）。
    *   `[终端: Linux Bash]` *💡 提示：按 `P` 键按CPU排序，按 `M` 键按内存排序。*
*   `lsof -p <pid>`: 查看某个进程打开了哪些文件（排查恶意进程）。
    *   `[终端: Linux Bash]` *💡 解释：lsof是"list open files"的缩写，用于查看进程打开的文件、网络连接等。*

### 2. 文件与权限
*   `ls -al`: 查看所有文件及权限（包括隐藏文件 `.` 开头）。
    *   `[终端: Linux Bash]` *💡 解释：ls是"list"的缩写，-a显示所有文件（包括隐藏文件），-l显示详细信息（权限、所有者、大小、时间等）。*
*   `chmod 755 file`: 修改权限。
    *   `[终端: Linux Bash]` *💡 解释：755表示拥有者可读写执行(7)，其他人可读执行(5)。*
*   `chown root:root file`: 修改文件所有者。
    *   `[终端: Linux Bash]` *💡 解释：chown是"change owner"的缩写，格式为 `chown 用户名:组名 文件`，这里将文件所有者改为root用户和root组。*
*   `find / -name "*.php" -mtime -1`: 查找24小时内被修改过的PHP文件（查Webshell必备）。
    *   `[终端: Linux Bash]` *💡 提示：Webshell通常是新上传的文件，此命令效率极高。*
*   `cat /var/log/secure | grep "Failed"`: 查看SSH登录失败日志（查爆破）。
    *   `[终端: Linux Bash]`

### 3. 用户管理
*   `cat /etc/passwd`: 查看所有用户（注意看uid=0的用户，除了root还有谁？）。
    *   `[终端: Linux Bash]` *💡 提示：攻击者可能创建 `toor` 这样的账号并修改uid为0来获取root权限。*
*   `last`: 查看最近登录成功的用户IP。
    *   `[终端: Linux Bash]`

### 4. 常规操作与Shell脚本
*   **文件操作**:
    *   `cp file1 file2`: 复制文件。
    *   `mv file1 file2`: 移动/重命名文件。
    *   `rm file`: 删除文件（`rm -rf` 递归强制删除，危险操作）。
    *   `mkdir -p dir1/dir2`: 创建多级目录。
*   **文本编辑**:
    *   `vi file`: 进入vi编辑器（按 `i` 进入编辑模式，`Esc` 退出编辑，`:wq` 保存退出）。
    *   `echo "content" > file`: 写入内容到文件（覆盖）。
    *   `echo "content" >> file`: 追加内容到文件。
*   **Shell脚本基础**:
    *   **创建脚本**: `vi script.sh`，开头添加 `#!/bin/bash`。
    *   **执行脚本**: `chmod +x script.sh && ./script.sh`。
    *   **示例脚本**（检查系统负载）:
        ```bash
        #!/bin/bash
        load=$(uptime | awk '{print $10}' | sed 's/,//')
        if (( $(echo "$load > 1.0" | bc -l) )); then
            echo "系统负载过高: $load"
        else
            echo "系统负载正常: $load"
        fi
        ```

## 🛡️ 常见安全设备认知 (面试必问)

你需要清楚这些设备部署在哪里，主要防什么。

### 1. 流量防御类

#### WAF (Web Application Firewall - Web应用防火墙)
*   **部署位置**：Web服务器前方。
*   **功能**：专门拦截HTTP/HTTPS流量中的攻击，如SQL注入、XSS、WebShell上传。
*   **面试题**：WAF和防火墙的区别？
*   **答**：传统防火墙防端口/IP（4层），WAF防Web攻击内容（7层）。

#### Firewall (传统防火墙/下一代防火墙 NGFW)
*   **部署位置**：网络边界。
*   **功能**：控制端口访问（如只开放80/443），做NAT，防DDoS。
*   **类型**：
    *   包过滤防火墙：基于IP、端口、协议过滤。
    *   状态检测防火墙：跟踪连接状态。
    *   应用层防火墙：基于应用协议过滤。

#### IDS / IPS (入侵检测/防御系统)
*   **IDS (Detection)**：旁路部署，只报警不阻断（像摄像头）。
*   **IPS (Prevention)**：串联部署，发现攻击直接阻断（像保安）。
*   **检测方法**：
    *   基于特征：匹配已知攻击特征。
    *   基于异常：检测异常行为模式。

### 2. 主机防御类

#### HIDS (Host-based IDS) / EDR (Endpoint Detection and Response)
*   **安装位置**：每台服务器/终端上（Agent）。
*   **功能**：监控文件变更、注册表修改、异常进程启动。
*   **知名产品**：360天擎、奇安信天眼、CrowdStrike。
*   **检测能力**：
    *   文件完整性监控。
    *   进程行为监控。
    *   网络连接监控。
    *   注册表监控。

### 3. 管理与审计类

#### 堡垒机 (Bastion Host / Jump Server)
*   **功能**：运维人员必须先登录堡垒机，再跳转去目标服务器。用于**审计运维操作**（录像、记录命令），防止删库跑路。
*   **核心功能**：
    *   身份认证（多因素认证）。
    *   权限控制（基于角色的访问控制）。
    *   操作审计（录像、日志）。
    *   会话管理（会话锁定、强制断开）。

#### VPN (虚拟专用网络)
*   **功能**：加密通道，让员工在外网能安全访问公司内网资源。
*   **协议类型**：
    *   SSL VPN：基于HTTPS，使用方便。
    *   IPSec VPN：基于IP层，性能较好。
    *   L2TP VPN：基于二层协议。
*   **安全措施**：
    *   多因素认证。
    *   客户端安全检查。
    *   访问控制列表（ACL）。

## 🔒 数据安全技术

### 1. 数据加密
*   **加密算法**:
    *   **对称加密**: AES-256（速度快，适合大数据）。
    *   **非对称加密**: RSA-2048（用于密钥交换、数字签名）。
    *   **哈希算法**: SHA-256（用于数据完整性校验）。
*   **应用场景**:
    *   **传输加密**: HTTPS (TLS/SSL)。
    *   **存储加密**: 数据库透明加密 (TDE)。
    *   **密钥管理**: 使用HSM (硬件安全模块) 保护密钥。

### 2. 访问控制
*   **模型**:
    *   **DAC (自主访问控制)**: 用户自主管理权限。
    *   **MAC (强制访问控制)**: 系统强制实施安全策略。
    *   **RBAC (基于角色的访问控制)**: 按角色分配权限（企业首选）。
*   **实施**:
    *   **最小权限原则**: 仅授予必要的权限。
    *   **权限分离**: 关键操作需多人配合。
    *   **定期审计**: 检查权限分配是否合理。

### 3. 数据脱敏
*   **脱敏技术**:
    *   **静态脱敏**: 数据存储前脱敏。
    *   **动态脱敏**: 查询时实时脱敏。
*   **脱敏方法**:
    *   **替换**: 将敏感信息替换为虚构数据。
    *   **掩码**: 保留部分信息，如手机号 `138****8888`。
    *   **截断**: 截断敏感信息。
    *   **哈希**: 对敏感信息进行哈希处理。
*   **应用场景**:
    *   **测试环境**: 使用脱敏数据进行测试。
    *   **开发环境**: 防止开发人员接触真实数据。
    *   **数据分析**: 在保护隐私的前提下进行分析。

## 🔥 防火墙与WAF配置

### 1. Windows防火墙配置

#### 基本防火墙管理
```powershell
# 查看所有防火墙规则
Get-NetFirewallRule | Select-Object DisplayName, Direction, Action
```
*   *💡 解释：此命令列出所有防火墙规则的名称、方向和动作，帮助了解当前防火墙配置状态。*

#### 添加入站规则
```powershell
# 添加入站规则（允许特定端口）
New-NetFirewallRule -DisplayName "Allow HTTP" -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow
```
*   *💡 解释：创建一个入站规则，允许外部访问本机的80端口（HTTP服务），常用于Web服务器配置。*

#### 添加出站规则
```powershell
# 添加出站规则（阻止特定IP）
New-NetFirewallRule -DisplayName "Block Malicious IP" -Direction Outbound -RemoteAddress "1.2.3.4" -Action Block
```
*   *💡 解释：创建一个出站规则，阻止本机访问恶意IP地址1.2.3.4，用于防止恶意软件外联。*

### 2. Linux防火墙配置 (iptables)

#### 基本规则管理
```bash
# 查看当前规则
iptables -L -n -v
```
*   *💡 解释：查看当前iptables防火墙的所有规则，包括链、规则、数据包统计等信息。*

#### 添加允许规则
```bash
# 添加规则（允许SSH）
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
```
*   *💡 解释：在INPUT链添加一条规则，允许TCP协议的22端口（SSH）流量进入，确保远程管理可用。*

#### 添加阻止规则
```bash
# 添加规则（阻止特定IP）
iptables -A INPUT -s 1.2.3.4 -j DROP
```
*   *💡 解释：添加一条规则，丢弃来自IP地址1.2.3.4的所有数据包，用于阻止恶意IP访问。*

#### 保存规则
```bash
# 保存规则
iptables-save > /etc/iptables/rules.v4
```
*   *💡 解释：将当前iptables规则保存到文件，确保系统重启后规则仍然有效。*

### 3. WAF规则配置

#### ModSecurity规则示例
```yaml
# ModSecurity规则示例
SecRule REQUEST_URI "@rx \.php$" \
    "id:1001,phase:1,deny,status:403,msg:'PHP files not allowed'"
```
*   *💡 解释：创建一个ModSecurity规则，阻止所有访问.php文件的请求，用于防止PHP文件执行攻击。*

```yaml
# ARGS: 检查所有请求参数
SecRule ARGS "@rx union.*select" \
    "id:1002,phase:2,deny,status:403,msg:'SQL Injection detected'"
```
*   *💡 解释：创建一个规则，检测请求参数中是否包含SQL注入特征（union select），防止SQL注入攻击。*

### 4. WAF绕过思路

#### 大小写混合
*   **方法**: `UnIoN SeLeCt` (针对老旧WAF)
*   *💡 解释：通过大小写混合绕过基于字符串匹配的WAF规则。*

#### 编码绕过
*   **方法**: URL双重编码 (`%2527` -> `%27` -> `'`)
*   *💡 解释：通过URL编码绕过WAF的字符过滤。*

#### HTTP参数污染 (HPP)
*   **方法**: `?id=1&id=2` (WAF检第一个，后端用第二个)
*   *💡 解释：通过重复参数名，让WAF只检查第一个参数。*
